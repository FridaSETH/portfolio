User Interface for Alert Handling on Autonomous Fleet Monitoring

Nøkkelord: autonomous shipping, fleet monitoring, user interface, alert handling, situational awareness, human-in-the-loop, cognitive load, maritime safety, operator interface, interactive design

This project presents a user interface for monitoring alerts in autonomous maritime fleets, improving situational awareness, prioritizing critical alerts, and reducing information overload across multiple screens. Developed with SFI Autoship and NTNU, it supports better decision-making and incident management for an imaginary fleet in Trondheim Fjord.

Fellow students: Caroline Foss Nersnæs and Stian Tangen Olerud

INTRODUCTION

Monitoring autonomous fleets poses challenges for handling multiple alerts and maintaining situational awareness. This project designs a human-centered interface that prioritizes critical alerts, reduces information overload, and supports effective decision-making, with a design based on the OpenBridge design system.

THE CHALLENGE

Current maritime interfaces are designed for single vessels and struggle with fleet monitoring, especially when handling multiple alarms and distinguishing between navigation and engineering issues.

APPROACH

We narrowed the scope to a single fleet in the Trondheim Fjord and focused on alert handling. Drawing on case studies like the Viking Sky accident and research on Human-In/Out-of-the-Loop systems, we prioritized situational awareness, minimizing information overload, and cognitive economy. We also incorporated principles and UI components from the OpenBridge design system, adapting them to better highlight critical alerts.

OUTCOME

The interface is designed across three screens in the control room, each supporting a different level of operation and enhancing fleet monitoring and decision-making:

  • Overview Screen: Wall-mounted display showing the fleet map, active alarms, and ongoing tasks—providing shared situational awareness.
  • Operator Desktop Screens: Interactive workstations where operators can explore alerts in detail, track vessel timelines, and distinguish between problem types.
  • Operator Tablets: Mobile devices delivering task-specific information, allowing operators to manage and resolve issues efficiently while on the move.

Together, these screens form a cohesive system that improves incident tracking, visualizes alerts over time, and reduces the risk of overlooked alarms.

OVERVIEW SCREEN

Displays a live fleet map with vessels, speed, and direction, plus filters for overlays like buoys and lighthouses. A minimap provides context, and color-coded alarms highlight severity. Designed for shared situational awareness, it reduces clutter and supports quick decision-making.

OPERATOR DESKTOP SCREEN

Interactive desktop for detailed alert handling and vessel timelines. Alerts show severity, type, and operator engagement, while dropdowns and a swipe-to-view gesture provide deeper detail without overload. Supports fleet-wide situational awareness and confident decision-making.

OPERATOR TABLETS

Mobile interface for task-specific management. Issues display a timer, animated cues, and nested details to guide attention and prevent overload. Enables operators to efficiently resolve problems while staying in the loop with autonomous systems.
